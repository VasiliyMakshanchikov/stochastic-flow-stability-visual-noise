# Markov Chain Model для анализа потока изменений числа людей

## 1. Назначение модели

Данная модель предназначена для описания динамики количества людей в кадре как **дискретной марковской цепи**. Она позволяет:

- строить матрицу переходов между состояниями (число людей);
- вычислять стационарное распределение;
- выделять бинарные события (изменение числа людей на пороговую величину и более);
- сравнивать поведение модели на основе «истинных» данных (ground truth) и данных, полученных с помощью компьютерного зрения (CV), подверженных шумам.

Модель используется в рамках исследования устойчивости стохастических моделей потоков к ошибкам визуального измерения. Основной исследовательский вопрос: **как искажения CV влияют на оценку матрицы переходов и стационарного распределения, а также на выделение событий потока**.

## 2. Что такое марковская цепь в контексте потока

**Состояние** — количество людей в кадре в дискретный момент времени (кадр, typically 1 секунда).  
Обозначение: `S(t) ∈ {0, 1, 2, …, K}`, где `K` – максимальное наблюдаемое значение.

**Марковское свойство**:  
`P(S(t+1) = j | S(0), S(1), …, S(t)) = P(S(t+1) = j | S(t) = i)`.  
То есть вероятность перехода в следующее состояние зависит только от текущего состояния.

**Матрица переходов** `P` размера `(K+1)×(K+1)`:

$P[i][j] = P(S(t+1)=j | S(t) = i)$

Сумма по каждой строке равна 1.

**Стационарное распределение** `π` (если цепь эргодична) – вектор вероятностей, удовлетворяющий:

$\pi = \pi*P, \sigma_{\pi_i} = 1$

`π_i` – доля времени, которую система проводит в состоянии `i` при длительном наблюдении.

## 3. Событие потока

В исследовании поток событий определяется как **изменение числа людей на пороговую величину и более** за один кадр (секунду).  
Бинарная последовательность событий:

$E(t)=1, если |S(t+1)-S(t)| >= treshold$

$E(t) = 0,$  инчае

По умолчанию `threshold = 3`, но параметр может варьироваться для анализа чувствительности.

## 4. Как модель используется для сравнения «истина – шум»

1. **Истинные данные**: из `mall_dataset/mall_gt.mat` с помощью функции `get_frame_timestamps` (significant_events=False) получаем список изменений счёта. Функция `extract_states_from_events_list` восстанавливает непрерывную последовательность `S_true(t)`.

2. **Зашумленные данные**: через Grounding DINO на искажённых изображениях получаем `S_noisy(t)`.

3. По каждой последовательности строим:
   - матрицу переходов `P_true` и `P_noisy`;
   - стационарное распределение `π_true`, `π_noisy`;
   - бинарные события `E_true`, `E_noisy` (при разных порогах).

4. Сравниваем:
   - числовые метрики (MAE, MSE, KL-расхождение) между матрицами;
   - графики (heatmap матриц, баровые диаграммы стац. распределения);
   - влияние порога на способность компенсировать шум.

## 5. Описание функций Python

Все функции реализованы в файле `markov_chain_model.py`.

| Функция | Назначение |
| ------- | ----------- |
| `build_transition_matrix(states, max_state=None)` | Построение матрицы переходов по массиву состояний. |
| `stationary_distribution(P, tol=1e-12, max_iter=10000)` | Вычисление стационарного распределения (итерационный метод). |
| `event_sequence(states, threshold=3)` | Формирование бинарной последовательности событий. |
| `build_binary_transition_matrix(events)` | Построение матрицы переходов 2×2 для бинарной цепи событий. |
| `extract_states_from_events_list(events_list, total_frames=None)` | Восстановление полной временной последовательности состояний из списка событий (из `get_frame_timestamps`). |
| `compute_transition_metrics(P_true, P_noisy, metric='mae')` | Вычисление метрики различия между матрицами. |
| `simulate_markov_chain(P, initial_state, n_steps, seed=None)` | Генерация синтетической последовательности по матрице переходов. |
